Ruby Association Grant 申請: Pure Skills

プロジェクト概要

Pure Skills は、急成長中の AI エージェント能力フレームワークと Model Context Protocol（MCP）ツール群の世界に対して、Ruby エコシステムに信頼に足る、Ruby らしい入口を与えるライブラリの提案である。今日、この分野のほぼすべての参照実装・SDK・実運用フレームワークは Python か TypeScript で書かれている。Ruby の不在は技術的な限界ではなく、基盤の欠落である。すなわち、AI エージェントの「skill」— エージェントが呼び出せる個々の能力 — を第一級の Ruby オブジェクトとして扱い、明快な構文で宣言し、版管理し、改訂間の差分を取り、構造を検証し、事後に監査できるようにする Ruby らしいツールキットが、現在存在しない。Pure Skills はこの欠落を、kairos-chain — RubyGems で公開済みの、MIT ライセンス・約 30,000 行の既存 Ruby gem で、AI エージェント向けの blockchain 裏付けの知識・skill 管理システムとして実際に動作しているもの — から、実証済みで領域中立な部分を抽出・一般化することで埋める。ゼロから新しい基盤を作るのではなく、実運用で既に機能しているものを、kairos-chain 本体とは独立に任意の Ruby 開発者が採用できる単独の再利用可能ライブラリへと蒸留する。

このプロジェクトは、Ruby がこの問題に適しているという特定の主張の上に立つ。Ruby の構造的自己言及性 — 単一の DSL と AST 表現が、「妥当な skill とは何か」というメタレベルの規則と、実際の skill 定義というベースレベルの実体の両方を記述できること — により、これは既存の Python / TypeScript 実装の移植ではなく、真に Ruby らしい様式となる。Ruby のブロックは、下で実オブジェクトグラフを構築しながら、DSL を自然な宣言構文として読ませる。オープンクラスは、fork せずにサードパーティ gem がその DSL を拡張することを許す。そして RubyVM::AbstractSyntaxTree をはじめとする実行時に検査可能な AST と内省機構により、専用パーサではなく言語自身の内省を用いて skill 定義の検証・差分・変換ができる。この三つの性質をこれほどきれいに併せ持つ主要言語は他にない。

成果物（Deliverables）

本助成は、プロジェクト範囲で定義された三つの構成要素それぞれにつき、ちょうど一つずつ具体的な成果物を生み、そのすべてを OSI 承認のオープンソースライセンスである MIT ライセンスで公開する。

  1. Pure Skills DSL と AST コンパイラ。 Pure Skills の宣言構文を実装する単独の Ruby gem。skill 定義を正準的な Ruby 抽象構文木（AST）にコンパイルし、declare・validate・diff・transform の各操作を公開の文書化された API として提供する。テストスイート、API ドキュメント、そして能力の構造を検査し、内部整合性を確認し、版間で比較し、通常の Ruby データとして安全に書き換える方法を示す実例を同梱する。MIT ライセンスで公開する。
  2. Ruby MCP サーバーツールキット。 Model Context Protocol の実装本体、能力を MCP クライアントへ公開するための tool registry、そして stdio と Streamable HTTP の両輸送層を備えた再利用可能ライブラリ。Ruby 開発者が輸送層とプロトコル層を毎回作り直すことなく、規格準拠の MCP サーバーを立てられるようにする。MCP 仕様に対する適合テストとサーバー雛形の実例を含む。MIT ライセンスで公開する。
  3. ハッシュ連鎖監査履歴ライブラリ。 すべての skill 定義とその後の変更を記録する、追記専用・ハッシュ連鎖の構成要素。あらゆる Ruby 製エージェントシステムに対し、どの時点でどの能力が存在し、どう変更されたかについての軽量で独立に検証可能な履歴を与える。kairos-chain 固有の業務ロジックにも、特定のエージェント編成方式にも依存しない、フレームワーク非依存の部品とする。MIT ライセンスで公開する。

各成果物は単独でも有用である。Ruby 開発者は、能力宣言のために DSL/AST コンパイラだけを、規格準拠サーバーのために MCP ツールキットだけを、検証可能な変更履歴のために監査履歴ライブラリだけを、他の二つを要さずに採用できる。

日程（Schedule）

プロジェクトは助成期間にわたって遂行し、Ruby Association 標準の二つの報告節目に沿って報告する。

中間報告（目標: 助成期間の中間点）。 中間報告までに、Pure Skills DSL と AST コンパイラを kairos-chain から抽出し、領域固有の前提を除いて一般化し、テストスイートと基本ドキュメントを備えた初期動作リリースとして公開する。MCP サーバーツールキットの初期設計と骨組み — tool registry と stdio 輸送層の第一次実装を含む — にも着手している。中間報告には動作するコード、テスト結果、後半期間に残る作業の明確な記述を含める。

最終報告（目標: 助成期間の終了時）。 最終報告までに三つの成果物すべてを完成させる。DSL/AST コンパイラは完全なドキュメントと実例を備えて確定。MCP サーバーツールキットは stdio と Streamable HTTP の両輸送層とプロトコル適合テストを備えて完成。ハッシュ連鎖監査履歴ライブラリは抽出・一般化され、独立にパッケージ化される。最終報告には全テストスイートの結果、公開済み gem 成果物、完全な API ドキュメント、そして各成果物を Ruby コミュニティが独立に採用する方法の要約を含める。

両報告とも、実装中に判明した範囲の調整を含めて進捗を透明に記録する。これは設計と実装を不可分の活動として扱う反復的開発の方針に沿うものである。

本助成のもとで生産されるすべての成果物 — Pure Skills DSL と AST コンパイラ、Ruby MCP サーバーツールキット、ハッシュ連鎖監査履歴ライブラリ — は、OSI 承認のオープンソースライセンスである MIT ライセンスで公開される。これは上記のすべての構成要素に、中間・最終いずれの報告段階でも、例外なく適用される。MIT ライセンスの選択は意図的である。派生元である kairos-chain のライセンスと揃えること、商用・非商用を問わず Ruby プロジェクトによる下流採用を最大化すること、そしていかなる成果物も特定企業の製品・基盤・事業モデルへの依存や優遇を持たないことを保証するためである。オープンソース基盤を作る開発者であれ、社内の独自ツールを作る開発者であれ、あらゆる Ruby 開発者が三つの構成要素の任意の組み合わせを、摩擦なく、ライセンス条件への全き信頼をもって採用できることを意図している。

MCP とエージェントツール群のエコシステムへの信頼に足る Ruby らしい入口を与えることで、Pure Skills は、急成長するソフトウェア基盤領域における Ruby 言語とそのコミュニティの立ち位置に資することを目指す。三つの構成要素 — DSL/AST コンパイラ、MCP サーバーツールキット、監査履歴ライブラリ — はいずれも領域中立で、応用領域を問わず、エージェントツール・社内自動化・規格準拠 MCP サーバーを作るあらゆる Ruby 開発者に一般に適用できる。本助成が資金を出すのは製品や事業ではない。Ruby コミュニティ全体がその上に築ける基盤である。

プロジェクト名: Pure Skills

Ruby は、Model Context Protocol（MCP）を中心に、より広くは AI エージェント能力フレームワークを中心に急成長しているツール群のエコシステムから、ほぼ姿を消している。この分野のほぼすべての参照実装・SDK・実運用フレームワーク — プロトコルサーバー、tool registry、能力スキーマ、エージェント編成ライブラリ — は Python か TypeScript で書かれている。これは Ruby の根本的な限界ではなく、利用可能な基盤の欠落である。AI エージェントの能力 — エージェントが呼び出せる個々の「skill」— を第一級の Ruby オブジェクトとして扱う Ruby らしいツールキットは現在存在しない。すなわち、明快な構文で宣言でき、コードのように版管理でき、どう進化したかを差分で見られ、構造的制約に対して検証でき、エージェントが実際に何をなぜ行ったかを事後に確定するために監査できるもの、である。表現力のある DSL 設計、メタプログラミング、読みやすい構文という Ruby 自身の強みは、まさにこの種の基盤に自然に適合するのに、その強みを再利用可能な能力定義ツールキットへと一般化するものが現在何もない。Pure Skills はこの欠落を埋めるために提案される。

Pure Skills は kairos-chain — RubyGems で公開済みの、MIT ライセンス・約 30,000 行の既存 Ruby gem で、AI エージェント向けの blockchain 裏付けの知識・skill 管理システムとして動作しているもの — から抽出・一般化される。基盤をゼロから作るのではなく、実証済みで領域中立な部分を単独の再利用可能ライブラリへ蒸留する。具体的に、フレームワークは三つの構成要素を提供する。第一に、skill 定義を正準的な Ruby 抽象構文木へコンパイルする Pure Skills DSL。declare・validate・diff・transform の各操作を公開し、能力の構造を、不透明な設定ファイルではなく通常の Ruby データとして、検査・整合性確認・版間比較・安全な書き換えの対象にする。第二に、プロトコル実装本体、能力を MCP クライアントへ公開する tool registry、stdio と Streamable HTTP の両輸送層を備えた再利用可能な Ruby MCP サーバーツールキット。Ruby 開発者が毎回プロトコル層を再実装せずに準拠サーバーを立てられるようにする。第三に、すべての skill 定義とその変更を記録する追記専用・ハッシュ連鎖の監査履歴部品。特定のエージェントフレームワークや業務ロジックから独立に、どの能力が存在しどう変更されたかの軽量で検証可能な履歴を、あらゆる Ruby 製エージェントシステムに与える。

Ruby がこの問題に特に適している理由は、その構造的自己言及性にある。単一の DSL と AST 表現が、メタレベルの構成物（妥当な skill とは何かという規則）とベースレベルの能力定義（実際の skill の実体）の両方を、二つの別個のスキーマ言語を要さずに記述できる。これが可能なのは、Ruby のブロックが、下で実オブジェクトグラフを構築しながら DSL を自然な宣言構文として読ませるからであり、オープンクラスが同じ DSL をサードパーティ gem が fork なしに拡張することを許すからであり、実行時に検査可能な AST（RubyVM::AbstractSyntaxTree と関連の内省機構）が、専用パーサではなく言語自身の内省で skill 定義の検証・差分・変換を可能にするからである。この三つの性質をこれほどきれいに併せ持つ主要言語は他になく、それゆえこの様式の Ruby らしい実装は、既存の Python / TypeScript 実装の移植ではなく真に新規である。

本助成のもとで生産されるすべての成果物は領域中立な Ruby 基盤 — DSL/AST コンパイラ、MCP プロトコルツールキット、監査履歴ライブラリ — であり、OSI 承認の MIT ライセンスで公開され、特定企業の製品や事業モデルへの依存も優遇も持たない。DSL ツールキット、MCP サーバーフレームワーク、監査履歴部品は、応用領域を問わず、エージェントツール・社内自動化・規格準拠 MCP サーバーを作るあらゆる Ruby 開発者にとって、それぞれ単独でも有用である。MCP とエージェントツール群のエコシステムへの信頼に足る Ruby らしい入口を与えることで、Pure Skills は、その上に築かれる特定の事業や製品ではなく、Ruby コミュニティと、急成長するソフトウェア基盤領域における Ruby の立ち位置に資することを目指す。
