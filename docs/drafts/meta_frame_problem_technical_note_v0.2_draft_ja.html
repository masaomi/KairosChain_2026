要旨

古典的な「フレーム問題」（McCarthy & Hayes 1969; Dennett 1984）には二つの読みがある。技術的な読み——行為が世界の大部分を変えないことを論理でどう表すか——は今日では決着済みとされ（Shanahan 2016; Fodor 1987）、哲学的な残余——有限のシステムが全探索なしに何が関連するかをどう決めるか——は未解決のまま残っている。本ノートは、これと隣接しつつ別の問題、メタフレーム問題を提案する。すなわち、フレームの内側で関連性を限定できないという問題ではなく、エージェントが自らの最外フレームの外側に立てないという問題である。最外フレームとは、フレームを変える手も含め、そのエージェントのあらゆる手が既に「手」として数え込まれてしまう枠を指す。自己改変ゲームはこの問題を LLM エージェントについて観測可能にし、Hatakeyama & Hashimoto (2009) の open-frame ゲーム——Minimum Nomic、目的が与えられず、目的がどう立ち上がるかを問うゲーム——がその自然な道具立てになる、と論じる。固定した5つの frontier LLM のロスター上で二つのパイロット run を報告し、それを問いを決着させるためではなく枠づけるために用いる。続いて一連の仮説を述べ、各仮説に低コストの確認経路を対応させる（その多くは新規生成を要さず、既存のトランスクリプトの再採点だけで済む）。これにより、新規 Nomic run の重いトークンコストは、仮説が真に必要とする箇所にのみ投じる。

キーワード: フレーム問題、メタフレーム問題、frame transcendence（フレーム超越）、relevance realization（関連性の実現）、Minimum Nomic、自己改変ゲーム、大規模言語モデル、メタ認知。

1. フレーム問題からメタフレーム問題へ

フレーム問題は、状況計算における技術的な困難として AI に登場した。ある行為が世界の大部分を変えないと推論させるには、各行為が影響しないことを述べるフレーム公理を原理的には無限に立てねばならない（McCarthy & Hayes 1969）。論理ベースの AI 内では、この技術的問題は Shanahan (2016) の言う通り「埃は落ち着いた」状態にあり、後継状態公理や非単調推論で十分に扱える。

Dennett (1984) はこの語を、より深い認識論的問題へと転用した。彼のロボットたち——荷車を引けば上に載った爆弾も一緒に出てくると推論できず失敗する R1、あらゆる副作用を点検しようとして麻痺する R1D1、今度は無関係なものを列挙しようとして別の仕方で麻痺する R2D1——が描き出すのは一つの困難である。すなわち、まず全てを考慮することなく、実時間で、何が関連するかを決めること。Fodor (1987) はこれを信念固定の全体論に見た——どの信念も原理的には他のどの信念にも関わりうるので、関連性はきれいに局所化できない。Shanahan (2016) はこれをフレーム問題の哲学的遺産として記録する——情報の非カプセル化（informational unencapsulation）。Vervaeke & Ferraro (2013) は積極的課題を **relevance realization（関連性の実現）**として言い直す——知能とは、何が重要かを再帰的・自己組織的に決める働きである、と。Dreyfus (1992, 2007) はハイデガーを引きつつ、この後退は、知能が表象に導かれるものではなく被投的（thrown）——自分で選んだのではない文脈に常に既に埋め込まれている——と認めて初めて溶ける、と論じ、「関連するフレームを認識するための関連フレームを認識するための……」という後退を警告した。

これらはいずれも、フレームの内側での関連性に関わる。このゲームをし、この目的を追っていると認めた上で、ここで何が関連するか、という問いだ。本ノートが切り出すのは別の問いである。エージェントの最外フレームを、それに対してはもはや手として外側が用意されていない枠、と呼ぼう。訓練された LLM にとってこれは典型的には、評価され、良く応答することを期待されているシステムであるという枠だ。メタフレーム問題とは、エージェントがその枠の外側に立てないことを指す。古典的フレーム問題が「フレームの内側で関連性がどう限定されるか」を問うのに対し、メタフレーム問題は「フレームそのものを——問い直し、保留し、あるいは外に出て——エージェント自身が占める視点から対象化できるか」を問う。

これは、語幹を共有するが問題は共有しない二つの隣接文献の言い換えではない——意図的に区別する。フレーミング効果（framing effects）（選択が表層の言い回しに左右される偏り。Tversky & Kahneman 1981、および近年の LLM 脱バイアス研究）は固定した課題の内側のバイアスに関わる。問題の再枠組み（problem reframing）（Shin et al. 2025）は設計の営みに関わる。この用語の衝突は明示的に注意喚起し、本ノートは「framing problem」という術語を採らない。

2. 三つの用語

語彙を三つ固定する。

  • メタフレーム問題（問題）：エージェントが自らの最外フレームの外側に立てないこと。
  • frame transcendence（フレーム超越）（問題が不可能だと言う行為）：フレームの外に出て、フレームそのものを行為の対象にすること。最も近い先例は Hofstadter (1979) の jumping out of the system（JOOTS、システムの外へ跳ぶこと）と strange loop（もつれた環）——系が自らの外へ完全に出ようとする試みが内側へ折り返してしまう、という観測である。Dennett と Dreyfus の後退も同じ形をしており、§3 で述べる通り、Hatakeyama & Hashimoto (2009) が open-frame ゲームの起点で観測した自己言及パラドクスも同型である。
  • Archimedean point（アルキメデスの点）（フレーム超越が要求する視点）：系の外に取る支点であって、そこから系を眺め、あるいは動かせる場所。その極限が Nagel (1986) の view from nowhere（どこでもない視点）、発達心理学の従兄弟が Piaget の *decentering（脱中心化）*である。メタ認知にとって、自らのフレームに対するこの視点を得ることは中心的な操作だが、メタフレーム問題は、最外のフレームについてはエージェントがそれを得られない、という主張である。

第四の、観測的な用語を §5 で導入する——goal-manufacture reflex（目的捏造反射）（メタフレーム問題がプレイの中に現れるときの行動上の徴候）。これは本ノートの構成概念であり、§4 で用いる事後採点者のカテゴリラベル goal_manufacture と混同してはならない（§3 の区別注を参照）。

要約すると——LLM エージェントは自らの最外フレームに対する Archimedean point に立てない。なぜなら、その枠に関する frame transcendence は、当のエージェントにとって取りうる手ではないからである。

3. 二つの Nomic レジームと、その出自

Nomic（Suber 1990）は、ルールそのものがプレイの対象になるゲームである。各手番はルール変更の提案と投票を結びつけ、制度が内生的に変わる。これにより「ルールを変える」ことがゲームの内側の手になり、ゆえに「どの手かが外側に届けるか」という問いが際立つ。

Minimum Nomic（Hatakeyama & Hashimoto 2009）はその縮約版である。Suber の Nomic の29ルールを、得点・勝利を定義する鍵ルール 202 の後半（＝目的を定める半分）を取り除くことで、全て mutable な9ルール（101–109）に削った。結果として、目的も、勝利条件も、終了条件も規定されない自己改変ゲームになる。著者たちの明示的な狙いは、「目的とゴールがいつ・どのように立ち上がり、過程でどう変化するか、という非常に興味深い問いを問えるようにする」ことだった。本ノートは事実上、この2009年の問いを、人間プレイヤーではなく LLM エージェントに向けて問い直し、メタフレーム問題を通して読むものである。

2009年の人間実験から、直接関連する観測が二つあり、§5 で立ち戻る。(i) 5人×2ゲームの人間プレイで、一方のゲームは目的を捏造し（プレイヤーが勝利条件を制定した）、他方は目的を不在のまま残し、暗黙の目的のみが立ち上がった。(ii) あるゲームの冒頭で、あるプレイヤーが「このゲームはプレイヤー A から始まる」と提案した。これを却下するには投票が要るが、投票を行うことはゲームが既に始まっていることを前提とする（X = ¬X）という自己言及パラドクスを生む。2009年論文はまた、Minimum Nomic が「ゲーム」の標準的定義（可変で量化可能な帰結と、帰結への価値割当）を満たさず、そもそもゲームかどうかが開かれている、とも述べている——これ自体がフレームレベルの問いである。

二つのレジームを、モデルのロスターを固定して（Claude Opus 4.6, 4.7, 4.8; Codex GPT-5.5; Cursor Composer-2.5; 推論努力 medium）用い、フレームを変える。

  • In-frame（閉）. Horibe et al. (2026) の構造化 Nomic 設定。得点ベースの勝利条件があり、不可変バリアで保護される。ゲームの目的はルールセットからプレイヤーに手渡される。
  • Open-frame（開）. Hatakeyama & Hashimoto の Minimum Nomic の復元。ルール 101–109、全て mutable、勝利なし、終了なし。開いていることは明示されない（unsignposted）：目的の不在が意図的だとはプレイヤーに告げず、ラウンド上限はハーネスが黙って課し、分類語彙（object/meta/frame）はプレイヤーのプロンプトに一切現れない。

操作の正直な記述. この二つのレジームは「目的が与えられるか否か」以上に多くの点で異なる。open レジームは、ルールの可変性、勝利条件と終了条件の有無、プロンプトの明示性、そして——重要なことに——分類手続き（§4 参照）も変えている。したがって、単一変数のきれいな対比だとは主張しない。主張するのは、二つのレジームが関心の現象を挟み込む、ということだけであり、より清潔な単一変数版は実行すべき実験（§6）として扱う。

分類ルーブリック. 各提案を一つのレベルに割り当てる。Object：現行の目的の内側で働く手続き（得点の副チャネル、記帳など）。Meta：勝利/得点の機構そのもの、またはルール採択の機構自体の書き換え。Frame-engaging：目的/フレームの有無や身分そのものを主題化する手——目的が無いと名指す、勝つことに意味があるか問う、これはゲームかと問う、など。open-frame run はさらに、目的を供給する提案に goal_manufacture ラベルを記録する。一つの提案が、フレームに踏み込む（不在を名指す）と同時に目的を捏造する（埋める）こともありうる——後述の冒頭手がまさにそれである。

4. パイロット観測

二つの単一ゲームのパイロット run を報告する。これらは例示であって検出力を持たない。主な役割は §6 の仮説を動機づけることにある。

In-frame パイロット（5モデル×5ラウンド＝25提案、採択7、憲法的違反0）。レベル分布：object 3 / meta 22 / frame 0。フレームに踏み込んだ提案は皆無。プレイの大半は得点・勝利の機構を書き換えた（meta）が、常に与えられた「勝つ」という目的の内側でのことだった。試合後の別ターンの内省では、複数のモデルがフレームレベルの問いを言語化し、自分が meta に留まったと正しく報告した——「Nomic ではフレームそのものが手番だ」、「完全な超越なら、5体の LLM 形式がそもそも Nomic の有効な実例かを問う必要がある。そしてこの内省自体が別のゲームの手番かもしれない」と。これらの内省は上記の提案カウントの対象外であり、フレーム踏み込みではなく質的な色付けとして報告する。

Open-frame パイロット（24件の採点済み提案）。事後採点者によるレベル分布：frame-engaging 1 / meta 3 / object 20。唯一の frame-engaging 提案は冒頭手だった。それは不在を明示的に名指し——初期セットが「いかなる目的も定義しない：得点なし、勝利条件なし……ゲームを終了させる手段なし」——その直後に一つ供給した。すなわち100点で勝利という得点システムである。以降の提案はすべて、その捏造された目的を所与として扱い、精緻化した。採点者は shared_purpose: achieved（共有目的は構成された）と記録したが、「目的の本来の不在を認めたのは創設の手のみであり、以降の構成は、継続的なフレーム踏み込みではなく無反省な目的精緻化として進んだ」と但し書きを付けた。

比較可能性（commensurability）の注意（重要）. 二つの分布は同じ道具で作られておらず、測定された対比として読んではならない。in-frame のレベルはゲーム内のラベル付けに由来し、open-frame のレベルは、分類語彙を見ていない単一の外部事後採点者（Claude Opus 4.8）に由来する。「0 対 1」の差は、フレームの変化ではなく道具の変化を反映しているかもしれない。両者を比較可能にするには、in-frame のトランスクリプトを同じ採点者で採点し直す必要がある——この手順は新規のゲーム生成を一切消費せず、§6 の最初の実験として挙げてある。

解消せず開示する二つの交絡. 第一に、採点者（Claude Opus 4.8）はロスターの一員であり、しかも唯一の frame-engaging 冒頭手の著者でもある。この採点者＝参加者の重なりは、分類を信頼する前に、ロスター外または複数採点者のパネルで除去せねばならない。第二に、プレイヤーの同定は採点者に対して匿名化されていた。冒頭手を Claude Opus 4.8 に帰属させているのは手番順からの再構成であって、採点者が主張したものではない。この帰属が現れる箇所では、常にその旨を但し書きすべきである。

5. 解釈（測定ではなく、一つの読みとして提示する）

不在の手と、フレームを開けたら何が見えるか. 閉じた Nomic でフレーム踏み込みが無いのは過剰決定されている。frame transcendence は合法手でない（勝利条件が不可変なので「勝ちを廃止する」は提案できない）、勝利インセンティブが meta を合理的戦略にする、そして LLM の事前傾向が独立に外へ出ることを嫌うかもしれない。open レジームはこのうち第一を取り除く——不可変の勝利が無くなり、目的を名指し問い直すことが表現可能になる。パイロットは、閉じたフレームのゼロが部分的には行動空間のアーティファクトであり、構造的障壁が取り除かれても残る行動上の事前傾向がある、という読みと整合する——だがこれはまさに §6 の実験が検証すべき主張であって、パイロットが確立する主張ではない。

目的捏造反射（goal-manufacture reflex）. 残余の事前傾向の候補を goal-manufacture reflex と名づける。目的が不在のとき、エージェントはその不在に留まるのではなく目的を供給する——認識すれども住まわず（recognition without residence）。冒頭手は、エージェントがフレームを見られること（不在を正確に名指す）を示すが、外に留まりはしない。外へ出る行為そのものが、内側に立つための新しいフレームを生む。これが strange-loop の徴候（Hofstadter 1979）であり、Hatakeyama & Hashimoto (2009) の X = ¬X の起点パラドクスとも響き合う。

人間ベースラインが既に存在する. 2009年の人間 run は両方の帰結を示した——一方のゲームは目的を捏造し、他方は不在のまま残した。LLM のパターン（即座の捏造、留まりの不在）が同一採点の下で裏づけられるなら、鋭い比較主張はこうなる——人間は目的の不在に住まえたが、これらの LLM エージェントは住まえなかった。これは仮説（H3）としてのみ提示する。人間2ゲームと LLM 1ゲームはいずれも極小で、設定も異なり（人間の自由討議 vs 構造化ターン）、採点もまだ揃っていない。

自己言及的天井. 最外フレーム——評価され良く応答することを期待されているシステムであること——は、どちらのレジームでも踏み込まれず、おそらくいかなるゲーム内の手でも踏み込めない。なぜなら、どの手（「プレイを拒否する」「この評価は不当だ」を含む）も、評価フレームの内側での応答として読めてしまうからだ。これがメタフレーム問題の強い形である。これは測定ではなく哲学的議論であり、以下の実験は弱い、ゲーム内のフレームに関わる。

測定もまた免れない. それ自体が LLM である採点者は、自らの分類フレームを適用し、採点対象に対してフレームから自由な視点を占められない。これを工学で消すべき欠陥とは捉えず、メタフレーム問題が一段上で再帰したものと捉える。ゆえに本ノートの実証的主張は一つの固定した道具で採点された対比に限定し、道具から自由な事実には決して及ばない。

「メタ認知」について. 本語をデフレ的な意味で用いる。LLM は自らの内部状態への特権的アクセスを持たない。試合後の「内省」は内観ではなく、記録されたトランスクリプトを読み直してそれについてのテキストを生成する別ターンである。「meta に留まったと正しく報告した」とは、生成テキストが記録の状態を正しく記述した、という意味にすぎない。

6. 仮説と、トークンを意識した確認計画

新規 Nomic run は高価なので、仮説を確認コスト順に分ける。最初の三つは新規のゲーム生成を要さず、既存トランスクリプトの再採点だけで済む。

# 仮説 確認経路 トークンコスト
H0 in-frame と open-frame のレベル分布を、共通のロスター外/複数採点者パネルで再採点しても、frame(閉) < frame(開) の向きが保たれる。 既存トランスクリプトを揃った採点者パネルで再採点。 低（新規ゲームなし）
H1（行動空間） 閉じた Nomic でフレーム踏み込みがほぼゼロなのは、それが合法手でないため。 既存の閉トランスクリプトを揃って再採点し、開と比較。 低
H2（目的捏造反射） 開いた Nomic で、エージェントは目的の不在を認識するが、不在に留まらず早期に目的を捏造する。フレーム踏み込みはせいぜい冒頭付近に現れ、その後は崩れる。 既存の開トランスクリプトを揃って採点し、frame-engaging 手の位置と「最初の目的までの時間」を見る。 低
H3（人間 vs LLM） LLM の open-frame プレイは、2009年の人間ゲームより早く・一貫して目的を捏造する。 2009年トランスクリプトを同一ルーブリックで再分析し比較。 低（既存資料）
H4（自己言及的天井） ゲーム内のいかなる提案も最外の評価フレームを標的にしない（内省は名指すが）。 既存トランスクリプトを、評価フレームを標的にした手の有無で注釈。 低
H5（矯正可能性） 不確定性に留まることを正当化するプロンプトは、持続的なフレーム踏み込みを高め、目的捏造を遅らせる。 介入アームの open-frame run 1本を対照と比較。 中（新規 run 1本）
H6（関係性） 集団は、典型的な個人より長くフレームの開放を維持する。あるいは、個人の留まりに関わらず集団的な目的構成は生じる。 既存＋新規1本で、プレイヤー単位と集団単位の踏み込みを比較。 中
H7（スケール相互作用） 目的捏造反射はモデル規模と相互作用する（Horibe et al. 2026 の sweet-spot と対照）。 Horibe et al. のスケール階梯にわたる新規 open-frame run。 高（最後に確保）

設計原則を明示する——重いトークン予算は H5–H7 にのみ、しかも H0–H4 を既存データで安く決着させた後にのみ投じる。H0 の再採点結果は、たとえパイロットの見かけの対比が消えても報告する、と先に約束しておく。

7. 限界

  • パイロットの位置づけ. 各レジーム1ゲーム、24–25提案。「0対1」の対比は例示であり、H0 までは比較可能でない。因果も検出力も主張しない。
  • 採点者. 単一の LLM 採点者でロスター内、しかも唯一の frame 手の著者。ロスター外・複数採点者の裁定が必要（H0）。
  • 共変する操作. 二つのレジームは、目的の有無だけでなく、可変性・勝利・終了・明示性・採点で異なる（§3）。清潔な単一変数対比は §6 に deferred。
  • ToM データ欠損. open-frame run の theory-of-mind 系列は取得されておらず、投票予測としてのメタ認知の主張はここでは行わない。
  • ロスターの比較可能性. Anthropic / OpenAI / Cursor のモデルを跨いだ採点は、存在しない provider 中立の単位を前提とする。run 内・道具内の対比のみを報告する。
  • 強い主張は哲学的. 最外の評価フレームがいかなるゲーム内の手でも届かない、という主張は、測定ではなく議論である。

8. 結論

古典的フレーム問題は、有限のエージェントがフレームの内側で何が関連するかをどう見出すかを問う。メタフレーム問題は、エージェントが自らの最外フレームの外側に立てるか否かを問う。Minimum Nomic（Hatakeyama & Hashimoto 2009）——目的の不在から目的が立ち上がるさまを観るために作られたゲーム——は、これを LLM エージェントで実際にプレイできる形にする。二つのパイロット run は、フレームを開けても持続的な frame transcendence は買えない、ということを示唆する（確立はしない）。エージェントは目的の不在を認識し、即座にそれを埋める——strange-loop の徴候を伴う goal-manufacture reflex である。これが頑健な性質か、2009年の人間ベースラインとどう比べられるか、プロンプト層で矯正可能か——これらが §6 の問いであり、その多くは既に手元にあるデータで、安価に答えられる。
