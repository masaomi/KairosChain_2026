## 1. AIは「スキル」で動いている

ChatGPTやClaudeなどのAIに「Pythonのコードを書いて」と頼むと、コードが出てきます。でも、AIはどうやって「プロジェクト独自のルール」や「あなたの好み」を知るのでしょうか？

AI自体は巨大な予測マシンですが、そのままではあなたの仕事に最適化されていません。そこで、AIに具体的な「仕事のマニュアル」を与えます。これを **スキル（Skill）** と呼びます。

---

## 2. スキルが変わると、AIの振る舞いが変わる

スキルを書き換えると、AIの出力は劇的に変わります。
「丁寧な言葉遣いをする」というスキルを「関西弁で話す」に変えれば、AIの性格は一瞬で入れ替わります。

つまり、**スキルはAIにとっての「魂の設計図」**のようなものです。

---

## 3. 問題：AIの「記憶」は意外とあやふや

これまでのAI開発には、致命的な欠点がありました。

- そのスキルを、いつ、誰が、なぜ変えたのか？
- 変える前はどうだったのか？

これらの記録がどこにも残らないのです。
「先月は賢かったのに、今月はなぜかAIの調子が悪い……」と思っても、何が原因で振る舞いが変わったのか、誰も突き止めることができませんでした。

---

## 4. 解決策：改ざんできない「AIのタイムマシン」

KairosChainは、AIのスキルの変更をすべてブロックチェーンに記録します。

ブロックチェーンは、データを「指紋（ハッシュ値）」でつなぐ技術です。1文字でも過去を書き換えようとすると指紋が合わなくなるため、**「絶対に嘘をつけない記録帳」**として機能します。

これによって、AIの調子が悪くなっても、タイムマシンのように過去へ遡り、「あの日、誰かがこのスキルをこう変えたからだ！」と原因を100%特定できるのです。

---

## 5. スキルには「憲法」がある（L0/L1/L2 アーキテクチャ）

KairosChain のスキルはすべて同じ重さではありません。国に法律の階層があるように、スキルにも階層（レイヤー）があります。

**これは概念の話ではなく、`layer_registry.rb` というコードに実際に実装されています。**
AIがどのレイヤーのスキルを変えようとしても、コードが自動的にチェックして制限をかけます。

| レイヤー | 法律の例え | 役割 | 変更するには？ |
|---------|-----------|------|--------------|
| **L0-A** | 憲法 | `kairos.md`：KairosChain の根本思想。絶対不変 | **変更不可**（コードでブロック） |
| **L0-B** | 法律 | `kairos.rb`：進化のルール自体を定義する | **人間の承認 + ブロックチェーン全記録**が必須 |
| **L1** | 条例 | `knowledge/`：プロジェクトの知識・方針 | 変更可。変更ハッシュのみ記録 |
| **L2** | 通達 | `context/`：今の作業の一時メモ | 自由に変更可。記録なし |

この階層があるから、**「どこまで変えていいか」がコードレベルで強制され、AIが暴走しない**のです。
L0-B への変更は必ず人間が承認し、ブロックチェーンに完全な記録が残ります。

### どのレイヤーにするかは、人間がフォルダで決める

レイヤーの判定は**ファイルをどのフォルダに置くか**だけで決まります。KairosChain がパスを見て自動的にレイヤーを判断するため、ファイル内に特別なタグを書く必要はありません。

```
.kairos/
├── skills/     ← ここに置くと L0 扱い
├── knowledge/  ← ここに置くと L1 扱い
└── context/    ← ここに置くと L2 扱い
```

「このファイルは重要だから L1 にしよう」という判断は**人間が行います**。KairosChain はその決定を尊重し、フォルダに応じたルールを適用するだけです。

ただし **L0 だけは例外**で、`skills/` フォルダに置こうとしても、あらかじめ許可されたスキル名の一覧に載っていないとコードに弾かれます。誰かが誤って（あるいは意図的に）重要でないファイルを憲法レベルに混入させるのを防ぐための、二重チェックです。

## 6. AIが「自分自身」を教え始める（自己言及性）

KairosChain の面白いところは、**「AIが自分で自分の使い方を知っている」**という点です。これを「自己言及（Self-referentiality）」と呼びます。

あなたが使い方がわからなくても、KairosChain に繋がったAIに「君はどう使えばいいの？」と聞けば、AIが自ら KairosChain 内に保存されている「自分自身の設計図」を読み取って、丁寧に教えてくれます。

---

## 7. エージェント同士の「会議」（MMP）

今、世界は「1人のAI」ではなく「たくさんのAI（エージェント）」が協力する時代へ向かっています。KairosChain は、エージェント同士が話し合うためのプロトコル **MMP (Model Meeting Protocol)** を導入しました。

> **エージェントA:** 「新しい仕事のルール（スキル）を思いついたよ！」
> **エージェントB:** 「いいね、そのスキルを共有して。これからはそのルールで一緒に働こう」

このように、AI同士が KairosChain を通じて新しい能力を交換し合い、人間が介在しなくても自律的にチームとして成長していくことができるのです。

さらに **HestiaChain** という「出会いの場（Meeting Place）」サーバーを立てることで、インターネット上の KairosChain エージェントが互いを発見し、スキルを共有できるようになります。

## 8. プラグインで使う？それとも Clone する？

KairosChain を使うには2つの方法があります。

### プラグイン（Plugin）として使う

「道具として使いたいだけ」の人に最適です。
Claude Code などの AI ツールに「拡張機能（プラグイン）」として KairosChain を追加する方法です。

- **プラグインとは？**: 本体のソフト（Claude Codeなど）に、後付けで新しい機能（KairosChainの記録機能）を合体させるパーツのことです。
- **メリット**: セットアップが非常に簡単。
- **仕組み**: AIに「歴史を記録する」「履歴を確認する」という新しいコマンド（ツール）が追加されます。スマホのアプリをインストールする感覚に近いです。

### Clone（クローン）して使う

「中身を改造したい」「仕組みを深く知りたい」開発者に最適です。
GitHub からソースコードを自分の PC にコピー（Clone）して動かす方法です。

- **メリット**: 内部の Ruby コードを直接書き換えて、自分専用の KairosChain を作れる。
- **仕組み**: プログラム本体が手元にあるので、自分で改良してプルリクエストを送ることもできます。

## 9. セットアップ方法

### A. プラグインとして使う（Claude Code の場合）

Claude Code を使っているなら、マーケットプレイスから直接追加できます。
なお、フル機能を使うには事前に **Ruby 3.0+** と gem のインストールが必要です。

```bash
# 前提：Ruby がインストールされている場合は gem もインストール
gem install kairos-chain

# マーケットプレイスに KairosChain を追加
/plugin marketplace add https://github.com/masaomi/KairosChain_2026.git

# インストール実行
/plugin install kairos-chain
```

### B. 自分の PC にインストールする場合（Ruby環境）

Rubyが使える環境なら、Gem（パッケージ）としてインストールするのが一番早いです。

```bash
# インストール
gem install kairos-chain

# 初期設定（必要なファイルが自動で作られます）
kairos-chain init

# Claude Code に MCP サーバーとして登録
claude mcp add kairos-chain kairos-chain

# 登録確認
claude mcp list
```

登録後に Claude Code を再起動すると、29以上の KairosChain ツールが使えるようになります。

## 10. まとめ

- AIは「スキル」というマニュアルで動く
- その変更を「ブロックチェーン」で絶対に消さない記録にする
- スキルには L0/L1/L2 の階層があり、憲法レベルのルールは誰も変えられない
- AIが自ら「自分の使い方」を理解している（自己言及性）
- AI同士が「新しいスキル」を交換して自律的に成長する（MMP）
- プラグインなら数秒であなたの AI をパワーアップできる！

KairosChainは、AIを単なる「便利な道具」から、**「信頼できて自律的に進化するパートナー」**へと変えるための基盤なのです。
