Existing Approaches (SDD / ADD / TDD+Agents)

Design origin is human. AI is the implementer.
Top-down: from human intent to AI execution.
vs

AFD (Agent-first Driven Development)

Build Agent Tool ‚Üí Agent validates ‚Üí Extract API ‚Üí Add UI
Design origin is agent interaction. AI is the first consumer.
Bottom-up: from working tools to APIs to UIs.

Classify by Cognition: The Architectural Decision
API-able

Deterministic CRUD, queries, config management ‚Üí Extract to REST API

Partially API-able

CRUD core + optional AI modes ‚Üí Split: API path + Agent path

Agent-native

Requires LLM reasoning, multi-perspective analysis ‚Üí Keep as MCP tool

Principle 1
Agent-First
Design agent tools as the system's first interface
Principle 2
Validate through Interaction
Business logic validated by agent usage, not UI prototypes
Principle 3
Extract, Don't Design
APIs extracted from validated tools, not designed in isolation
Principle 4
Dual Interface
Final system serves both AI agents and human users
Principle 5
Classify by Cognition
Separate API-able (deterministic) from Agent-native (LLM) ops
Principle 6
Zero-Gap Deployment
Tool implementation = deployment. No staging needed.
